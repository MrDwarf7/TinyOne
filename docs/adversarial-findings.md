Probe 2 (Hostile artifacts): PASS — Both `artifact_rejects_huge_counts_before_accepting_program` and `artifact_rejects_invalid_integer_fields_and_file_size` passed. Probe 3 (Verifier stress):   PASS — All three verifier tests passed: `verifier_handles_dense_jump_graph_without_path_explosion`, `verifier_rejects_oversized_dense_jump_graph_before_stress`, `verifier_rejects_stack_depth_bomb`. Backward-jump analysis: the `visit` function (verifier.rs lines 274-305) uses a `seen: HashMap<usize, i64>` to record (pc → stack_depth) on first visit. For `[PushInt(0), JumpIfZero(0), Halt]` — PC0 is first visited with depth=0, placed in `seen` and `todo`. After processing PC0 (PushInt, depth becomes 1), the JumpIfZero at PC1 targets PC0 with depth=0 (after consuming the condition). When `visit` is called for PC0 with depth=0, `seen.get(&0)` returns `Some(0)`, depths match (0==0), so it returns `Ok(())` without re-queuing — no loop. If instead PC1 had pushed first then jumped back, the depth would be 1 vs 0 in `seen`, and line 295-299 immediately returns `Err("stack depth mismatch …")`. The backward-jump loop is safely terminated by the `seen` guard in all cases.
